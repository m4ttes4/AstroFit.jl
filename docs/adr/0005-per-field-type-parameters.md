# One type parameter per field

Every model struct declares one type parameter per field (`Gaussian1D{A,M,S}`, not `Gaussian1D{T}` with three `::T` fields), and `withparams` reconstructs a leaf with `constructorof(M)(fields...)` — no `promote`. Fields no longer have to agree on a type, so a leaf whose amplitude arrived from the parameter vector as a `ForwardDiff.Dual` beside two stored `Float64` constants constructs directly. This deletes `_ctorexpr`, `_isparamfield` and the field contract it documented, and twelve hand-written promoting constructors in the zoo; it also keeps fixed fields out of dual arithmetic during AD, which measured 5.8 μs against 8.0 μs on a gradient over a gaussian with one of three fields free.

Keeping the shared `T` plus promotion was rejected: it costs a promoting constructor per model, forces the package to explain why a `Float64` field is fittable while an `Int` field is not, and makes every fixed parameter carry the full dual width during differentiation for no benefit.

The cost is a breaking change for user models, and an intermittent one: a struct sharing a type parameter across fields keeps working while all of them are free, and fails with a `MethodError` from inside generated code the moment one is fixed and a gradient is taken. Mitigation is documentation (README "Extending AstroFit"); a compile-time diagnostic was considered and deferred — it cannot live in `@model`, which sees syntax rather than types and runs before `@fix`, so the only place with both the field types and the Free/Fixed split is the `@generated withparams`.
